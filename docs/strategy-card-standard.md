# Strategy Card Standard v1.0

> The quant Model Card. Three tiers of strategy documentation.

## Why

AI has Model Cards. Quant has nothing. GIPS reports fund performance to investors.
Tearsheets are marketing. Papers are non-reproducible. No standard answers:
"How was this discovered? How honestly was it validated? What fails?"

## Three Tiers

| Tier | Name | Audience | Trust | Generated by |
|------|------|----------|-------|-------------|
| 1 | **CARD** | Public (allocators, peers) | Safe to publish (no alpha) | `causal-edge validate --card` |
| 2 | **SPEC** | Private (auditors, deployers) | Contains alpha — never publish | Manual (code + params + data) |
| 3 | **METHOD** | Public (agents, researchers) | Safe to publish (teaches method) | Manual (SKILL.md / methodology) |

Each tier: YAML frontmatter (machine-readable) + markdown body (human-readable).

## Tier 1: CARD (Public Evaluation)

Auto-generated by `causal-edge validate --csv backtest.csv --card strategy-card.md`

### Required Fields (MUST)

```yaml
schema: strategy-card/1.0
tier: card
name: "Strategy Name"
date: YYYY-MM-DD

performance:
  sharpe: float
  lo_adjusted: float
  max_dd: float (negative)
  total_pnl: float

triangle:
  lo_adjusted: float  # ratio space
  ic: float           # rank space
  omega: float        # shape space

validation:
  framework: causal-edge
  verdict: PASS | FAIL
  score: "N/M"
  tests:
    dsr: {value, threshold, status}
    pbo: {value, threshold, status}
    # ... all 15 tests
```

### Recommended Fields (SHOULD)

```yaml
asset: TICKER
asset_class: crypto | equity | commodity | fx
direction: long_only | long_short | long_flat | market_neutral

risk:
  skewness: float
  hill_estimator: float
  cvar_var_ratio: float
  worst_year: {year, sharpe}

robustness:
  dsr: float
  pbo: float
  oos_is: float
  loss_years: int
  bootstrap_p: float

discovery:
  method: causal | correlation | fundamental | ml_only | hybrid
  K: int          # total trials for DSR honesty
  source: string  # e.g. "abel-cap"

backtest_days: int
limitations:
  - "Known limitation 1"
  - "Known limitation 2"
```

### Markdown Body

```markdown
# Strategy Name

## Summary
[What the strategy does — no implementation details]

## Metric Triangle
[Table: Lo / IC / Omega]

## Validation: VERDICT (score)
[Framework, profile, failures if any]

## Limitations
[Known failure modes, regime sensitivity]

## Discovery Provenance
[Method, K, source — if disclosed]
```

## Tier 2: SPEC (Private Reproduction)

NOT auto-generated. Written by strategy author. NEVER published for live strategies.

### Schema

```yaml
schema: strategy-card/1.0
tier: spec
card_ref: "./strategy-card.md"  # links to Tier 1

code:
  engine: path/to/strategy.py
  entry_point: "run_strategy(data) -> (pnl, dates, positions)"
  dependencies: [list of packages]

parameters:
  # The alpha — every param needed to reproduce
  parents: [list]
  clip: float
  min_train: int
  # ...

data:
  sources:
    - {provider: fmp, tickers: [...], start: YYYY-MM-DD}
  calendar: crypto_ffill_equity | trading_days | calendar_days

research:
  branch: git-branch-name
  results: path/to/results.tsv
  experiments: int
  keeps: int

reproduce:
  command: "python evaluate.py"
  expected_sharpe: float  # +/- tolerance
```

## Tier 3: METHOD (Public Methodology)

NOT auto-generated. Written as a skill or methodology document. Safe to publish.

### Schema

```yaml
schema: strategy-card/1.0
tier: method
skill_ref: "causal-alpha"  # link to methodology skill

axioms:
  - "Causal K < blind K → DSR honest (Pearl)"
  - "Look-ahead = invalid backtest (definitional)"

constraints:
  - "Multi-metric validation > single (Lo × IC × Omega)"
  - "Serial compounding > grid search"
  - "Explore = new information, not subtraction"

discovery_protocol: "references/discovery-protocol.md"
research_protocol: "references/experiment-loop.md"
validation_framework: causal-edge
```

## How Tiers Compose

```
METHOD (public)          SPEC (private)           CARD (public)
teaches discovery    →   full reproduction    →   evaluation summary

causal-alpha             strategy code            causal-edge validate
produces METHOD          produces SPEC            produces CARD

"How to fish"            "This specific fish"     "How big is the fish"
```

## What This Adds vs Model Card

| Feature | HF Model Card | Strategy Card Standard |
|---------|--------------|----------------------|
| Tiers | 1 | 3 (card + spec + method) |
| Discovery provenance (K, method) | No | Yes |
| Anti-gaming validation (triangle) | No | Yes |
| Privacy tiers | No (weights public) | Yes (spec private) |
| Methodology layer | No | Yes |
| Machine-readable | YAML frontmatter | YAML frontmatter (compatible) |
| Auto-generated | Partial | Tier 1 fully automated |

## Usage

```bash
# Generate Tier 1 from any backtest
causal-edge validate --csv backtest.csv --card strategy-card.md

# Generate Tier 1 for all strategies in project
causal-edge validate --card strategy-card.md

# Tier 2 and 3 are documentation — write them manually
```
